# Security And Safety Disclosure

COSMOS-CQA is a research-only public project stewarded by AI-Bio Synergy
Holdings LLC. This note summarizes how to report security, safety,
accessibility, and data-handling concerns without turning public project
surfaces into disclosure risks.

## Reporting Lanes

- Private vulnerability reports belong in the GitHub security advisory
  workflow when available, or by email to
  `cosmos-cqa-developer@ai-biosynergyholdings.com` with the subject
  prefix `COSMOS-CQA vulnerability report`.
- Public safety reports belong in the repository safety issue template
  only when the report does not include sensitive details.
- Public accessibility reports belong in the repository accessibility
  issue template only when examples, screenshots, or recordings are
  non-sensitive and shareable.
- General public research questions can use the contact route or public
  issue templates when no private data is involved.

## What To Keep Out Of Public Reports

Do not include credentials, private keys, session cookies, personal
health information, regulated data, restricted datasets, private source
code, private screenshots, deployment secrets, or medical/audiology
advice in public GitHub issues, pull requests, comments, or email
examples.

## Project Boundaries

COSMOS-CQA is not a production service, clinical tool, diagnostic system,
regulated workflow, or validated scientific decision system. The hosted
demo is a static, local-first research workbench. Security and safety
reports should be framed around the public portal, browser workbench,
shared package contracts, release artifacts, documentation, and public
claim boundaries.

## Stewardship Response

AI-Bio Synergy Holdings LLC reviews reports as capacity allows. The
project does not offer a bug bounty, service-level agreement, production
incident response commitment, or guarantee of private response. When a
report can be handled publicly, maintainers may create a sanitized issue
or pull request that avoids sensitive reproduction details.
