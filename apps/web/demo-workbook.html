Hosted demo manual

Demo workbook for the public Core Pack workflow.

Use this workbook to review the sample workbench as research infrastructure: inspect the Core Pack, exercise tile labeling, export evidence, and stay inside the public claim boundary.

Before you begin

Research-only boundary

The hosted demo is not a production decision system and does not publish validated cosmology diagnostics.

Public label policy

Public truth labels remain hidden in the visible workflow and public DOM text. Dev truth review is only for explicit local development mode.

Local-first data

The static portal has no account system. Downloads, bookmarks, and imports stay in the browser session unless you share them.

Evidence focus

Use exports to inspect contracts, provenance hashes, SBOM references, known limitations, and deterministic replay surfaces.

Twenty-minute walkthrough

  1. Open the hosted demo. Confirm the banner reports that the sample Core Pack is ready.
  2. Inspect Core Pack intake. Review manifest summary, tile passports, evidence references, SBOM references, and diagnostic references.
  3. Review the first tile. Change overlay and palette once, then confirm the caption and tile passport remain readable.
  4. Submit and undo a label. Add a brief note, submit a label, then undo it to confirm the session can recover from review mistakes.
  5. Create a bookmark. Use the bookmark control, reload the copied state URL, and confirm tile, overlay, palette, rate, loop, and caption settings return.
  6. Check diagnostic caveats. Read placeholder status and limitations before treating any diagnostic surface as evidence.
  7. Refresh report preview. Use the Reports panel to regenerate the validation preview before export.
  8. Export evidence. Download SBOM JSON, validation report JSON, session JSON, and evidence bundle JSON for local review.

What to record during review

Workflow state Demo URL, browser, viewport size, timestamp, and whether public mode or dev mode was active.
Core Pack evidence Manifest id, tile passport ids, evidence reference paths, SBOM reference ids, and provenance hash subjects.
Interaction evidence Tile navigation, overlay/palette changes, label submit/undo behavior, bookmark reload result, and export filenames.
Boundary notes Known limitations, diagnostic caveats, public truth-label hiding, and any accessibility or visual issues observed.

Local verification commands

npm --prefix apps/web run test:browser -- test/browser/hosted-demo.spec.mjs
npm --prefix apps/web run test:browser -- test/browser/portal.spec.mjs
npm --prefix apps/web run check:portal-deploy
npm --prefix apps/web run check

The focused hosted-demo test verifies public boundaries and evidence export behavior. The portal test verifies public pages, metadata, handoff, and responsive usability.

Review outcome

A completed workbook pass should leave you with a validation report JSON, optional SBOM JSON, optional session/evidence bundle JSON, and a short note describing any visual, accessibility, or research-boundary concerns. File implementation bugs as GitHub issues and route public research questions through the contact page.
